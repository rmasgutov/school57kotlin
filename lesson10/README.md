# Домашнее задание: Генерация документации через аннотации и рефлексию

**Дедлайн: 14 декабря, 23:59**

---

## Цель

Реализовать логику генерации человеко-читаемой документации для Kotlin-классов с использованием **аннотаций** и **рефлексии**. На основе метаданных, заданных через аннотации, программа должна во время выполнения выводить структурированное описание класса, его свойств и методов.

---

## Задача

Вам нужно **реализовать метод**:
```kotlin
fun generateDoc(obj: Any): String
```
в уже существующем объекте `DocumentationGenerator`, чтобы он возвращал текстовую документацию на основе аннотаций, расставленных по коду.

---

## Что уже есть

- Все аннотации (`@DocClass`, `@DocProperty`, `@DocMethod`, `@DocParam`, `@InternalApi`) уже объявлены.
- Объект `DocumentationGenerator` с заглушкой `generateDoc` уже создан.
- Набор unit-тестов уже написан и проверяет корректность работы.

Вам **не нужно**:
- Создавать аннотации
- Писать тесты
- Менять сигнатуры

Вам **нужно**:
- Реализовать логику `generateDoc` так, чтобы **все тесты проходили**.

---

## Формат вывода
Пример кода:
```kotlin
@DocClass(description = "Пользователь системы", author = "Alice", version = "1.2")
class User(
    @DocProperty(description = "Имя пользователя", example = "john") val name: String
) {
    @DocMethod(description = "Приветствует пользователя", returns = "Приветственное сообщение")
    fun greet(@DocParam(description = "Суффикс") suffix: String): String {
        return "Hello, $name $suffix!"
    }
}
```

Пример результата:
```
=== Документация: User ===
Описание: Пользователь системы
Автор: Alice
Версия: 1.2

--- Свойства ---
- name
  Описание: Имя пользователя
  Пример: john

--- Методы ---
- greet(suffix: String)
  Описание: Приветствует пользователя
  Параметры:
    - suffix: Суффикс
      Возвращает: Приветственное сообщение
```

---

## Что скрывать

- Все элементы (классы, свойства, методы), помеченные `@InternalApi`, **должны быть проигнорированы**.
- Автоматически сгенерированные методы `data class` (`toString`, `equals`, `hashCode`, `copy`, `componentN`) **не должны появляться** в документации.

---

## Требования к реализации

- Используйте `kotlin-reflect` для доступа к классу, свойствам и методам.
- Обрабатывайте:
    - Класс: `@DocClass`, `@InternalApi`
    - Свойства: `@DocProperty`, `@InternalApi`
    - Методы: `@DocMethod`, `@InternalApi`
    - Параметры: `@DocParam`
- Если у класса нет `@DocClass` → вернуть `"Нет документации для класса."`
- Если класс помечен `@InternalApi` → вернуть `"Документация скрыта (InternalApi)."`
- Поля и методы без аннотаций всё равно отображаются (без описания).
- Пустые секции (`Свойства`, `Методы`) не должны отображаться, если нет соответствующих элементов.
